1) bank_data.txt (सारा डेटा – लेवल‑वाइज़ ट्री)
यह एक JSON फॉर्मेट वाला TXT है (यानी सिंटेक्स JSON जैसा, पर फाइल एक्सटेंशन .txt है)।
इसमें पूरा ट्री है:

ऊपर: RBI

क्लिक करने पर उस नोड का सारा work (जैसे – कौन किस सेक्टर में क्या काम करता है, किस तरह के लोन, किस पर निगरानी आदि) नीचे खुल जाता है।

समझ गया – अभी जो प्रॉब्लम है, वो ये है कि सारे बच्चे लगभग एक ही लेवल में “गच्चा” हो रहे हैं, जबकि आपको proper rows चाहिए:

RBI → उसके नीचे A,B,C,D,E,F एक लाइन में → फिर उनके अपने बच्चे अगली लाइन में, वगैरह।
मैंने इसके लिए generate_bank_tree.py में दो बड़ी चीज़ें बदल दी हैं:
हर parent के बच्चों के लिए अलग row बनाया

पहले बच्चे सीधे parent के बाद आते थे, इसलिए कुछ cases में सब एक ही लेवल जैसा दिख रहे थे।
अब मैंने एक नया wrapper डाला है:
html


<div class="children-row">
    <div class="children-group">
        ...children...
    </div>
</div>
यानी हर parent के नीचे उसकी बच्चों की अलग row बनेगी।

CSS में connectors और spacing ठीक की

नया class: .children-row { margin-top: 50px; }
.children-group अब ऊपर से 30px नीचे शुरू होता है, और
horizontal व vertical लाइनें (::before, ::after) को फिर से position किया है, ताकि:
parent node से एक vertical line नीचे आए,
फिर उसके नीचे एक horizontal line बच्चों के ऊपर हो,
फिर उस horizontal line के बीच से vertical line नीचे की तरफ (जैसा org‑chart में होता है)।
इससे होना यह चाहिए:

Top row: सिर्फ RBI
दूसरी row: RBI के तुरंत नीचे उसके बच्चे:
A) Scheduled Banks, B) Co‑op Banks, C) DFIs, D) NBFCs, E) Payment & Market Infra, F) अन्य नियामक…
तीसरी row:
A के नीचे: A1, A2 …
B के नीचे: B1, B2 …
C के नीचे: C1, C2, C3, C4, C5 …
इत्यादि, हर group अपने parent के बिल्कुल नीचे दिखेगा, एक ही line में नहीं मिलेगा।
अब आप क्या करें
ये फाइल पहले से अपडेट है: generate_bank_tree.py
टर्मिनल / CMD में चलाएँ:
bash


python generate_bank_tree.py

RBI सबसे ऊपर,

आप जो बोल रहे हो, वो दो चीजें हैं:

Hierarchy power के हिसाब से दिखे – किसके ऊपर कौन है, देखते ही समझ आ जाए।
देख कर ही पता चले कौन ज़्यादा powerful है, कौन नीचे है – सिर्फ लाइन/लेवल से नहीं, टेक्स्ट से भी।
मैंने इसके लिए अभी दो काम किए हैं:

1) डेटा में “Power Level” जोड़ दिया

A1.1) PSBs, A1.2) Private, A1.3) Foreign, A1.4) RRB, A1.5) SFB → "power_level": 4

इसका मतलब:

Level 1 (Power 1) → सिर्फ RBI
Level 2 (Power 2) → पूरे सिस्टम के बड़े power blocks: Scheduled, Co‑op, DFIs, NBFC, Payment Infra, Regulators

2) ये HTML क्या करती है?
जब फाइल खोलेगे, तो:

Right side पर पहले से एक छोटा sample tree दिखेगा

RBI → Scheduled Banks → Commercial / Co‑op / DFIs / NBFCs / Regulators
हर node color + lines के साथ, proper hierarchy में।
हर box में “More” बटन – क्लिक करो तो उस बैंक/संस्था का काम detail में खुलता है।
Left side पर एक बड़ा textbox होगा – “यहाँ डेटा डालें (JSON फॉर्मेट)”

इसमें अभी sample JSON भरा होगा (ताकि structure समझ आ जाए)।
तुम अपना data बदलो, नया node जोड़ो, कुछ हटाओ – सब यहीं।
नीचे दो बटन:

“Tree बनाओ” →
textbox वाला डेटा लेगा, JSON parse करेगा और उसी के हिसाब से नया tree बना देगा।
“फिर से Sample डेटा लोड करो” →
अगर डेटा गड़बड़ हो जाए तो फिर से RBI वाला basic structure लोड कर देगा।
अगर JSON में गलती हो (कौमा / bracket वगैरह), तो नीचे लाल box में “JSON Error: …” लिख कर बता देगा, ताकि तुम्हें पता लगे कहां गलती है।

3) डेटा कैसे डालना है? (फॉर्मेट)
हर नोड ऐसा दिखता है:

json


{
  "title": "A1.1) Public Sector Banks (PSBs)",
  "subtitle": "सरकार की majority हिस्सेदारी",
  "sector": "Govt Owned Banks",
  "power_level": 4,
  "work": [
    "जन‑धन, DBT, PM Kisan जैसी योजनाएँ",
    "ग्रामीण क्षेत्रों में मजबूत नेटवर्क"
  ],
  "children": [ ... sub-nodes ... ]
}
title → बॉक्स के ऊपर नाम
subtitle → छोटा description
sector → किस sector का bank / संस्था
power_level → power के हिसाब से नंबर (1 सबसे ऊपर, 4 नीचे) – इससे देख कर भी समझ आए कौन ज़्यादा powerful है
work → list में उन पर “More” क्लिक करने पर जो detail दिखानी है
children → इसके नीचे वाले nodes
सबसे ऊपर root node RBI है, और उसके अंदर children में A, B, C, D, E, F जैसा स्ट्रक्चर है।
तुम चाहो तो:

बस RBI वाला node रखो और बाकी सब हटा कर अपना पूरा नया सिस्टम बना सकते हो,
या existing sample को ही modify कर सकते हो (जैसे और banks, institutions, sectors add करना)।
4) Power के हिसाब से समझ
Depth (ऊपर से नीचे) लाइन और रंग से hierarchy दिखती है,
साथ में कार्ड पर Power Level: N भी दिख रहा है – ताकि तुरंत अंदाजा हो कौन ऊपर authority है, कौन नीचे ऑपरेशनल।
5) अगला कदम तुम्हारे लिए
interactive_bank_tree.html को open करो।
Right side पर basic sample tree देखो – confirm करो कि layout / lines तुम्हें पसंद है।
Left side में sample JSON को अपनी ज़रूरत के हिसाब से बदलो:
और banks जोड़ो,
अलग regulators, NBFC types, आदि add करो,
हर पर “work” में जितनी detail चाहो डालो।
“Tree बनाओ” दबाओ – नया tree ready।
अगर तुम चाहो तो मैं तुम्हारे लिए exact exam‑ready full data (पूरा बड़ा JSON) भी बना सकता हूँ जिसे बस copy‑paste करके डालना है, तुम केवल “Tree बनाओ” दबाओगे और हो जाएगा।

ठीक है, अब बिल्कुल step‑by‑step साफ़ करते हैं।

1. Error का मतलब क्या है?
Error:

Expected ':' after property name in JSON at position 2496 (line 66 column 28)
इसका मतलब:

आपने JSON में कहीं property लिख दी है लेकिन उसके बाद : नहीं है
या
कॉमा, डबल‑क्वोट या bracket कहीं गलत है।
JSON के कड़े नियम:

हर key ऐसे लिखनी है: "title": "..."
कभी भी single quote ' नहीं, सिर्फ "
list / object में आख़िरी item के बाद , (comma) नहीं होना चाहिए
हर key के बाद तुरंत : होना ज़रूरी है।
Screenshot से लग रहा है कि आप ने F5 / F6 वाले हिस्से (IIBF, IDRBT) के पास कुछ एडिट किया और वहाँ या तो:

किसी string के अंदर newline तोड़ दिया,
या "something" "Security" की तरह दो string आ गईं, बीच में : / , नहीं है।

3. आगे से JSON एडिट करते समय छोटी सावधानी
कोई property ऐसे न लिखें:

json


"title" "कुछ"
हमेशा:

json


"title": "कुछ"
किसी array के आख़िरी element के बाद comma न लगाएँ:

json


"work": [
  "line 1",
  "line 2"   // यहाँ comma नहीं
]
अगर आप चाहो तो अगली बार जो भी नया data डालोगे, उसे पहले मुझे text में भेज दो, मैं उसे valid JSON बनाकर दे दूँ, ताकि आपको ये error दोबारा न झेलना पड़े।
